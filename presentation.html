Alarm Path (Priority)

FLAG_ALARM → GLD retries until CH ACKs → CH immediately forwards → MQTT → NestJS → EventLog CRITICAL < 1 second end-to-end target

Normal Data Path

Buffered in CH RAM → server sends SERVER_PULL_REQUEST every 10 s → CH responds with CLUSTER_BULK_DATA batch → NestJS processes batch

CH Heartbeat

CH_HELLO every 300 s → server updates topology registry (nodeId → clusterId → gatewayId path) → used to route targeted pull requests

AppFrame Header (10 bytes)

BytesFieldValue / Notes
0Magic0xAA — always first
1typeFlagsmsgType[5:0] + FLAG_ALARM_ACK[6] + EXT_POWER[7]
2–3SrcIduint16 big-endian node ID
4–5DstIduint16 big-endian, 0xFFFF = broadcast
6SeqSequence number
7PayloadLenByte count of payload
8…NPayloadN bytes (type-dependent)
N+1–N+2CRC16CRC16-CCITT-FALSE, big-endian

SENSOR_DATA Payload (7 bytes)

ByteFieldValues
0schema_versionAlways 1
1power_mode0=battery, 1=external
2operation_mode0=running, 1=training, 2=nulling
3–4battery_mVuint16 big-endian (e.g. 3800)

Classification

AI Output → Risk Level Pipeline

10 / 17
LOW
predicted_class == 0
OR
confidence < 0.70
Save reading · No alert
MIDDLE
class ≠ 0
AND
0.70 ≤ conf < 0.80
Save + EventLog WARNING
HIGH
class ≠ 0
AND
confidence ≥ 0.80
Save + EventLog CRITICAL + ACK

addReading() — Auto-Threshold Logic

if (aiClass !== 0 && confidence >= 0.80)
  riskLevel = 'HIGH'
  createEventLog(CRITICAL, 15-min dedup)
  sendAck(back to GLD via CH)

else if (aiClass !== 0 && confidence >= 0.70)
  riskLevel = 'MIDDLE'
  createEventLog(WARNING, 15-min dedup)

else
  riskLevel = 'LOW'  // silent

Why AI Classes, Not PPM?

  • MQ sensors don't output calibrated PPM by default
  • Cross-sensitivity: one sensor responds to multiple gases
  • Multi-sensor array fed to TFLite — better classification accuracy
  • Confidence score captures model certainty — reduces false alarms
  • 15-minute deduplication window prevents alert storms

15-Minute Deduplication

An alarm event is only created if no EventLog exists for the same device within the last 15 minutes. Prevents alert storms from continuously sensing nodes.

Alarm-to-ACK Flow

GLD detects gas (class≠0, conf≥0.80)
Sets FLAG_ALARM · Sends alarm frame
CH receives → priority queue → forwards immediately
Gateway → MQTT: gld/gw/{id}/up
NestJS: parse → riskLevel=HIGH → addReading()
EventLog CRITICAL created
NestJS → MQTT ACK → gld/gw/{id}/dn → CH → GLD ACK
Dashboard shows alert · Operator acknowledges
